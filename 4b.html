Avoid These

Common Pitfalls to Avoid

Module 02: Framework

1. Being Too Vague

Prompt กว้างเกินไป ได้ผลลัพธ์ที่ไม่ตรงจุดหรือตื้นเกินไป

❌ "เขียนโค้ดให้หน่อย" ✅ "เขียน JS function สำหรับ sort array"

2. Overloading with Tasks

ยัดหลายงานในคำสั่งเดียว ทำให้ AI สับสนและคุณภาพต่ำ

❌ "สร้าง API + DB + Test + Deploy" ✅ แยกเป็น 4 prompts

3. Ignoring Context

ไม่ให้ข้อมูลพื้นหลัง AI จึงตอบกว้างๆ ไม่ตรงกับ use case

❌ "แก้ bug นี้หน่อย" ✅ "ใน React + TypeScript..."

4. Not Specifying Format

ไม่บอกรูปแบบผลลัพธ์ ได้ output ที่ใช้งานยาก

❌ "ให้ข้อมูล users" ✅ "ให้เป็น JSON array"

5. Expecting Perfect First Try

คาดหวังผลลัพธ์สมบูรณ์ตั้งแต่ครั้งแรก ไม่ปรับปรุง Prompt

❌ "ใช้ Prompt เดิมซ้ำๆ" ✅ Draft → Refine → Loop

6. Not Validating Output

เชื่อ AI 100% โดยไม่ตรวจสอบ อาจได้ข้อมูลผิดหรือ Hallucination

❌ Copy-Paste ทันที ✅ Review & Test ก่อน

Remember: AI ไม่ใช่ Magic ✨ — ผลลัพธ์ขึ้นอยู่กับคุณภาพของ Input (Garbage In, Garbage Out) และ AI อาจ "Hallucinate" ข้อมูลที่ไม่ถูกต้องได้
