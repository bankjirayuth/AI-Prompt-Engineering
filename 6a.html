Iterative Prompt Improvement & Feedback Loops

Module 02: Structure & Techniques

What? — คืออะไร?

กระบวนการปรับปรุง Prompt แบบวนลูป: ร่าง -> ทดสอบ -> วิเคราะห์ -> ปรับแก้

Why? — ทำไม?

  • •Context: AI เข้าใจบริบทครบถ้วน ไม่ตกหล่น
  • •Accuracy: ลดข้อมูลมั่ว (Hallucination) เพิ่มความแม่นยำ
  • •Robustness: รองรับเงื่อนไขพิเศษ (Edge Cases) ได้ครอบคลุม

Use Cases (Dev)

  • •Create Boilerplate: สร้าง Template สำหรับ Project Structure
  • •Create Unit Test: สร้าง Template สำหรับ Test Cases
  • •Create Documentation: สร้างเอกสารสำหรับ API/System
1

Write

เขียน Prompt แรก (Draft) ใช้องค์ประกอบของ Prompt ที่ดีเป็น guideline

Identify Gaps
4

Refine

ปรับปรุง Prompt ตาม feedback แล้วกลับไปทดสอบใหม่

Iterate Again

Best Practices

Version Your Prompts

เก็บ history ของ prompt แต่ละ version เพื่อเปรียบเทียบ

A/B Test Variations

ทดสอบ prompt หลาย versions กับ input เดียวกัน เปรียบเทียบผลลัพธ์

Define Success Criteria

กำหนด "ตัวชี้วัด" ที่ชัดเจน เช่น ความถูกต้อง (Accuracy), รูปแบบถูกต้อง (Format)

Document Learnings

จดบันทึกสิ่งที่เรียนรู้ว่า prompt แบบไหน work/ไม่ work

Pro Tip: เริ่มจาก prompt ง่ายๆ ก่อน แล้วค่อยๆ เพิ่มความซับซ้อนทีละ step
