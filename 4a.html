Optimization Mindset

Key Principles of Prompt Optimization

Module 02: Framework

1. Clear Intent

คำนิยาม: การระบุเป้าหมายของ Prompt อย่างชัดเจน ไม่คลุมเครือ

หลักการ:

  • • บอก AI ว่าต้องการ อะไร และ ทำไม
  • • หลีกเลี่ยงคำที่ตีความได้หลายแบบ
  • • ระบุเชิงปริมาณ เช่น "ไม่เกิน 3 ข้อ"

2. Context Depth

คำนิยาม: การให้บริบทและข้อมูลพื้นฐานที่เพียงพอ

หลักการ:

  • • ใส่ Background ที่จำเป็น
  • • ระบุ Constraints และ Dependencies
  • • ให้ข้อมูลที่ AI ต้องใช้ในการตัดสินใจ

3. Iterative Refinement

คำนิยาม: การปรับปรุง Prompt อย่างต่อเนื่องจาก Feedback

หลักการ:

  • • Draft → Test → Analyze → Refine
  • • อย่าหวังผลเลิศจากครั้งแรก
  • • เก็บ Version ที่ใช้งานได้ดี

4. Output Engineering

คำนิยาม: การควบคุมรูปแบบและโครงสร้าง Output

หลักการ:

  • • กำหนด Format: JSON, Markdown, Table
  • • ใช้ Negative Prompts: บอกสิ่งที่ไม่ต้องการ
  • • ระบุความยาวและ Tone ที่ต้องการ

Key Insight: Prompt ที่ดีคือการ สื่อสาร ไม่ใช่แค่ สั่ง — ยิ่งให้ข้อมูลครบถ้วน AI ยิ่งเข้าใจและตอบตรงจุด
